# Search Plan 与项目推进建议｜区域总经理

## 一页结论
- **项目判断**：该搜索项目不能只用“找简历”的方式推进，必须同步做画像校准、市场反馈收集与业务方预期管理。
- **首要目标**：前两周拿到真实市场样本，回答三个问题——有没有人、这些人愿不愿意聊、业务方想要的人是否现实。
- **关键提醒**：如果两周后仍无高质量样本，不应只催顾问提速，而应反过来校准岗位、薪酬与汇报关系。

## 项目背景
- **项目**：区域总经理 Search Plan
- **岗位**：区域总经理
- **公司背景**：全国连锁零售集团，华南区域业绩承压
- **任务背景**：不仅要补位，还要顺带重整区域团队与经营节奏
- **项目目标**：输出 4 周 search plan 和 stakeholder 校准建议
- **目标行业**：零售, 连锁经营, 区域管理
- **工作地点**：广州
- **预算区间**：年包 180-250 万

## 项目成功定义
1. 在 2 周内完成岗位画像校准，并形成一版被业务方确认的 target spec。
2. 在 3 周内拿到首批高质量 shortlist，并沉淀明确的市场反馈。
3. 在 4-6 周内推动至少 2-3 位可进入深度流程的候选人。

## 建议项目节奏
### Phase 1｜Kickoff 与画像校准（Week 1）
- 与业务负责人对齐：必须项、加分项、不能妥协项分别是什么。
- 将岗位改写为“使命 / 90 天结果 / 成功标准 / 风险提示”结构。
- 完成目标公司池、关键词库、触达话术第一版。

### Phase 2｜市场验证（Week 2）
- 输出 20-30 位样本名单，并复盘命中率。
- 对高匹配样本做首轮触达，记录回复率与拒绝原因。
- 召开第一次 search review，决定是否调整画像、地点或薪酬。

### Phase 3｜重点推进（Week 3-4）
- 集中资源推进高质量候选人，不做平均用力。
- 统一面试判断模板，减少不同面试官标准漂移。
- 对业务方持续提供市场教育：人才池厚度、竞争情况、候选人顾虑。

### Phase 4｜Offer 预判与风险管理（Week 5+）
- 提前验证候选人动机、家庭因素、汇报线接受度、薪酬区间。
- 对高概率人选做背调式追问，减少后期翻车。
- 预判 counter-offer 与流程拖延风险，设置推进 owner。

## 周会建议议程
1. 本周新增样本数、shortlist 数、进入面试数
2. 候选人拒绝原因 Top 3
3. 业务方对样本的共识与分歧
4. 是否需要调整画像、薪酬、地点或汇报线
5. 下周重点推进名单与 owner

## 过程指标看板
- **样本命中率**：反映画像是否清晰。
- **触达回复率**：反映岗位故事是否成立。
- **初面通过率**：反映 shortlist 质量。
- **流程流失率**：反映流程体验与决策效率。
- **Offer 接受概率**：反映动机与竞争态势。

## 主要风险与对策
### 风险 1：画像过理想化
- **表现**：业务方每看一批样本都说“不够全能”。
- **对策**：强制排序必须项，明确只能保留前三项核心标准。

### 风险 2：面试标准不一致
- **表现**：不同面试官对同一候选人评价分裂。
- **对策**：统一评分维度，至少覆盖结果、能力、动机、文化四项。

### 风险 3：决策链路过长
- **表现**：候选人反馈慢、等待久、兴趣下降。
- **对策**：指定唯一推进 owner，并提前锁定关键决策节点时间。

## 顾问建议的下一步
1. 48 小时内完成 kickoff 信息补齐，尤其是岗位成功定义与不能妥协项。
2. 72 小时内交付首版 mapping 清单与搜索逻辑，供业务方快速校正。
3. 一周后必须召开第一次 search review，用真实样本而非抽象讨论做决策。

## 综合判断
- **项目复杂度**：中高
- **信心等级**：中
- **一句话建议**：把 search 当成咨询项目来管，才不会沦为低效催简历。
