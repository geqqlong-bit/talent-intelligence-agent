# Intake fields

Use this reference when converting a fuzzy request into a structured brief.

## Core fields
- `projectName`: short task name, usually role + goal
- `roleTitle`: target role title
- `companyContext`: employer stage, team context, business model, or search context
- `hiringBrief`: raw business need or JD summary
- `objective`: what deliverable the user wants
- `targetIndustry`: likely talent source industry
- `targetCompanies`: likely company pool
- `location`: geographic hiring market
- `salaryRange`: compensation band if known
- `templateId`: the closest workflow template

## Helpful optional fields
- seniority
- reportingLine
- urgency
- languageRequirement
- mustHaveSkills
- disqualifiers
- interviewStage

## Default posture
Infer practical defaults and keep moving unless the missing field would materially change the advice.
